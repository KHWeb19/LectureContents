* 해결책

1. 우선 ChracterManager를 만들어서 전체 캐릭터들을 관리할 수 있게 만들어야함
2. BossRaidManager를 만들어서 레이드를 선택하고 보스전을 구성할 수 있도록 만든다.
3. ChracterManager를 만든 이후 기존 사냥 시퀀스를 다시 분석해볼 필요가 있음
   -> 제일 생각 난해한 부분이 RequestObject 파트다.
      -> 보편적으로 처음 개발하는 3 ~ 5년차 개발자들은 여기까지 생각하기 힘듬
         -> 파생되는 상황: 코드 구조가 지저분해짐

   -> RequestObject를 도입함으로써 필요한 정보를 RequestObject에게 요청하고
      그 정보를 받아서 필요한 정보만 끄집어 사용하는 패턴을 취함
      -> 실제 Spring에서도 이와 같은 패턴을 앞으로 많이 사용하게 될 것임

   * 언제 써야 할까 ? (RequestObject 같은 형식을)
     -> 뭔가 데이터가 필요한데 구조가 지저분해진다는 느낌을 받으면
        필요한 정보만 전달해줄 수 있는 Manager 역할로써 RequestObject를 쓰면 된다.
        -> 그냥 지저분해질것 같다는 느낌이 들 때 쓰면 평타는 칠 수 있다.
